System Architecture

Comprehensive overview of the system structure and components

High-Level Architecture

The Transfinder Form Assistant follows a layered architecture pattern with clear separation of concerns:

Frontend Layer (React)

Location: frontend/src/

Components: Chat interface, voice input, form display, report visualization

Communication: REST API calls to Go backend

API Layer (Gin Framework)

Location: handlers/

Endpoints: Chat, complaint, form, SQL, voice, results

Responsibilities: Request routing, validation, response formatting

Service Layer

Location: service/

Services: ComplaintService, SQLService, VoiceService

Responsibilities: Business logic, external API integration

AI Service Layer

Location: ai/

Capabilities: SQL generation, form generation, HTML generation, chat responses

Model: Qwen3-Max via DashScope API

Data Layer

Storage: BadgerDB (key-value store) + SQL Server

Location: db/

Data: Chat history, complaint state, SQL files, voice profiles

Data Flow

Complaint Flow

1. User Request

User sends complaint message via chat interface

2. Request Detection

Handler detects complaint keywords and routes to complaint flow

3. Initialize Process

ComplaintService calls Ground Control API to initialize flow

4. Start Dialogue

Begin conversation with Ground Control dialogue API

5. Continue Dialogue

Multiple exchanges until dialogue is complete

6. Execute & Finalize

Execute final step and return outcome to user

Form Generation Flow

1. User Request

User requests form creation via natural language

2. AI Form Generation

AIService generates form JSON based on template and user request

3. HTML Generation

AIService generates HTML page from form JSON

4. Save & Return

HTML saved to products/ folder and JSON returned to user

Student Report Flow

1. User Request

User requests report via natural language

2. SQL Generation

AIService generates SQL query using reference SQL files

3. SQL Execution

SQLService executes query against SQL Server

4. Result Processing

Results saved as JSON, then HTML page generated

5. Return to User

SQL query and result info returned to user

External Integrations

  • Ground Control API: Dialogue-driven flow execution for complaint processing
    • Base URL: http://192.168.9.136:8000
    • Endpoints: Special flows, dialogue management
  • DashScope API: AI model access (Qwen3-Max)
    • Used for: SQL generation, form generation, HTML generation, chat responses
  • SQL Server: Student database
    • Connection: SQL Server driver
    • Database: team2_ent
